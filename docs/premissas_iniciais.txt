------------- Premissas -------------
Os procedimentos hoje correm tanto no simp como no siga. no futuro talvez seja possível uniformizá-los e vir do próprio sistema novo. Então deve ser possível gerar um novo número ou usar algum digitando ou puxando de outro sistema.
Cada tipo de documento pode ter campos específicos para gravar na base de dados pertinente. Os campos para tramitar são os comuns usados nas capas processuais.
O tipo de documento pode determinar o fluxo de trabalho.

------------- Fluxo? -------------

1) Protocolar - iniciar o processo com capa referenciada com anexos os documentos importantes para iniciar o processo em questão (diárias, pedido de férias, licenças,....) assinado todas as páginas diferentes da capa para garantir a autenticidade;
2) Passar para o interessado;
3) O interessado receber o procedimento para se pronunciar e assinar (com assinatura digital em todas as novas páginas juntadas ou preenchidas) e enviar para o próximo interessado;
4) seguir o ciclo até o último movimento para o arquivo.

A qualquer momento posso ver o pdf do movimento 1, 2, 3, 4... ou o documento todo devidamente carimbado, paginado e assinado.

ASSINAR

ASSISTENTE JURÍDICO

ASSISTENTE MATEMÁTICO

------------ Campos para tramitar -------------------

NÚMERO (vindo do sistema SIGA)
DATA (hoje por default)
INTERESSADO (pessoa que fez o pedido/reclamação)
ÓRGÃO/UNIDADE INTERESSADO (órgão da pessoa interessada, se particular, "a sociedade")
ASSUNTO (com autocomplete e botão "+" para adicionar temas(*))
SUBASSUNTO (quando houver)
SUBSUBASSUNTO (...) (quando houver)
RESUMO DA TRAMITAÇÃO: 
1) Primeira pessoa que juntou ou se pronunciou;
2) Segunda pessoa que juntou ou se pronunciou;
3 ) Terceira pessoa que juntou ou se pronunciou; ....
(*) na verdade submeter a COGI para acréscimo na árvore de assuntos do MPBA)

--- Construção de Documento e Juntada ---
1. Construção de documento: editor de texto, podendo inclusive chamar campos preenchidos nas telas de processo e/ou movimento... como o Gmail.
2. Juntada: anexar n documentos
